# Portfolio Common-Executable Decision

Date: 2026-07-16

## Decision

**Rejected. Do not combine the maintained, high-profit, and reversion profiles on the proposed common executable. Do not promote a portfolio or a new best.**

The default-off account-wide exposure guard compiled and passed its static contract checks, but a safety guard is useful only if the executable preserves each strategy's trading behavior. The proposed common source does not preserve the high-profit stream.

## Provenance finding

The old public candidate profile declared source `F254...`, but the later `+$1,915.83` report was generated by source `8D62...`. The trading inputs in the corrected `dgf_hp_control` snapshot are identical to the old candidate except for the evidence profile ID, run label, and source hash. The source label was stale; the implementations are behaviorally very different.

## Additional blocker

The historical high-profit profile and maintained money-ready profile both use magic number `26070402`. Running them together would mix position ownership and trade-history accounting. They cannot be attached together unchanged.

## Required next architecture

Keep each exact strategy source isolated. Port the same account-wide exposure guard into each source independently, assign unique magic numbers, and require a guard-off control reproduction before any interaction test. Then run a true combined MT5 account test with the guard enabled, followed by Model 4, yearly, rolling-window, execution-cost, and broker-variation gates.

No source in this comparison is approved for live trading. Real-account launch remains hard-locked.
